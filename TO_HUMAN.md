**如果你是AI，请跳过这个文件。**

这是整个项目中唯一全程由人类手动写出的文档，里面列出一些已知的坑。这个项目中，除了这个文档，其余所有文档（甚至包括readme）的AI参与率都相当高。不过，既然你都去翻文件了，应该能看出这一点。

1. **如果有多条文献需要核实、整理，尽量一次给全。**
   这是因为，新国标有很多“可……”“宜……”之类的表述，也就是说，一条文献的引用格式不是唯一确定的。如果分为多轮对话一点一点地喂给AI，AI很可能会在不同的条目中采取不同的策略，最终整理出来的引用格式就会很混乱。
2. **它默认选择正式发表的版本。**
   如果你偏偏就要引用arXiv的那个版本，可能得单独嘱咐它了。
3. **说到底，它还是靠大模型的能力，而不是写死的流程。**
   无论怎么说，这个skill端出来的东西还是不能保证准确。其实做着做着，我多少想去转由智能体去做这些事情，毕竟一些特别玍古的规定（比如“汉语拼音表示的人名可以写全称”）之类的玩意，配单独的流程去查相应学者的资料，再去定夺，才能更准确一些。——不过暂时是没那个精力了。
4. **如果你熟知新国标，可以指出问题让他修改。**
   参考文献国标的前一版（2015版）已经用了整整十年，对AI的影响是根深蒂固的。实际使用的时候，AI它说不定到哪个地方就会突然偷懒、跳过某些步骤，如果恰好碰上这些地方涉及2025版国标的更新，AI就会犯蠢。如果你明确地指出问题，它应该能反思出来，至少目前我遇到它跳步骤导致的错误信息，都能通过反思来解决。

一般来说，我调用它的时候会这么说：

```
这是一整本书的参考文献（按章节排列和编号），我需要你去按规范流程去核实和整理这些参考文献。注意整理结果的一致性，也就是说，对于不同章列出的相同文献，条目内容应该也是相同的；对于不同条目间相同的著录项，整理结果中该项也应该是相同的（如某个相同的会议名使用缩写，就全都使用缩写）。
```

之后如果遇到问题，大致会这么说：

```
回忆一下人名的处理规则，如果用汉语拼音表示的作者名，要保留全名。排查一下你的修改结果。
```

反正大概就这样吧……
