# 贡献指南

感谢您对 Academic Paper Verifier 项目的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 报告问题

如果您发现了问题或有改进建议，请提交 Issue：

1. 检查是否已有相似 Issue
2. 创建新 Issue 并描述：
   - 问题描述
   - 复现步骤
   - 期望结果
   - 实际结果
   - 相关文献示例

### 提交代码

1. Fork 本仓库
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 打开 Pull Request

### 改进文档

文档改进也是非常宝贵的贡献：
- 修正错误
- 补充示例
- 改进说明
- 翻译

## 开发规范

### 代码风格

- 保持简洁清晰
- 添加必要的注释
- 遵循既有格式

### 提交信息规范

```
<type>: <subject>

<body>

<footer>
```

Type 类型：
- `feat`: 新功能
- `fix`: 修复
- `docs`: 文档
- `style`: 格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

### 示例

```
feat: 添加专利文献类型支持

- 实现专利 [P] 类型著录格式
- 添加专利示例
- 更新文档

Closes #123
```

## 行为准则

- 尊重他人
- 接受建设性批评
- 关注社区利益

## 联系我们

如有疑问，请通过 Issue 与我们联系。

感谢您的贡献！
