# 学术文献验证

## 概述

本技能用于验证学术文献的准确性，包括确认论文是否存在、核实关键信息（作者、标题、发表源、年份等），并依据 **GB/T 7714-2025《信息与文献 参考文献著录规则》** 生成规范的参考文献著录格式。

**核心原则：专事专办，按文献类型匹配对应要求，不堆砌信息。**

---

## 工作流程

### 步骤1：分析指令

首先分析用户发来的指令：
- **单条参考文献**：直接按后续步骤处理
- **多条参考文献**：制定计划，逐条依次处理

**重要说明：**处理多条文献时，必须对每条文献完整执行步骤2的所有子步骤（2.1至2.8），包括所有中间输出（查证表格、格式模板、必备项表格、修改后表格、单条引用），然后再处理下一条文献。禁止省略中间输出或跳过步骤。

### 步骤2：处理单条参考文献

**每条文献都要完整执行以下所有子步骤（2.1至2.8），并输出所有中间结果。**

#### 2.1 提取关键字

从用户提供的参考文献信息中提取关键字：
- 作者（主要责任者）
- 题名（标题）
- 发表源（期刊/会议/出版社等）
- 发表年份
- 其他已知信息（卷期、页码等）

#### 2.2 信息收集

使用网络搜索获取该文献的完整信息：

```
web_search 搜索关键词：论文标题 + 作者 + 发表年份
```

**搜索策略**：
- 优先使用精确匹配搜索
- 如无法获取，适当使用模糊搜索
- 尝试多个搜索引擎和学术数据库

#### 2.3 信息匹配验证

判断搜索结果是否与用户提供的文献一致：
- 比对作者姓名（注意拼写变体）
- 比对题名（注意中英文差异）
- 比对发表年份和来源
- 注意排除拼写相近但内容不同的文献

只保留可信的来源，输出查证表格：

| 序号 | 文献题名 | 查证链接 | 链接类型 | 备注 |
|:----:|----------|----------|----------|------|
| 1 | 题名1 | [链接1](URL) | 会议官方/期刊官网/预印本 | 正常/⚠️无会议链接 |

#### 2.4 识别文献类型与载体标识

根据搜索结果识别文献类型和载体类型，详细的标识代码见 `references/gbt7714-2025-core-rules.md`，速查见本文档末尾的**文献类型标识速查**。

**载体标识选择原则：**
- 如果文献类型为期刊[J]、会议[C]等，且存在纸质版本，则优先采纳纸质资源（不带/OL），即使是通过互联网获取的
- 仅在仅存在电子资源或明确需要标注电子载体时，才使用/OL标识
- 预印本[PP]、数据集[DS]、网页[EB]等天然电子资源类型使用/OL标识

**注意：如果同时存在纸质资源和电子资源，则优先采纳纸质资源。**

#### 2.5 获取引用格式模板

根据文献类型和载体标识，从 `references/citation-formats.md` 中获取对应的格式模板：

**示例格式模板：**

| 文献类型 | 格式模板 |
|----------|----------|
| 期刊论文[J] | 主要责任者. 题名: 其他题名信息[J]. 刊名, 年, 卷(期): 页码. 永久标识符. |
| 会议录[C] | 主要责任者. 题名: 其他题名信息[C]//会议名称, 会议年份: 引文页码. 永久标识符. |
| 预印本[PP/OL] | 主要责任者. 题名: 其他题名信息[PP/OL]. 版本. 出版平台(创建日期)[引用日期]. 获取路径. 永久标识符. |

**说明：需输出该格式的必备项说明，标注哪些是必备项。**

#### 2.6 收集素材与元数据

根据文献搜索结果和步骤2.4识别的文献类型与载体标识，仔细对照 `references/citation-formats.md` 中对应文献类型的著录项目，尽量完整地收纳所有素材和元数据，为后续排查做好铺垫。

**收集原则：**
- 作者名：尽量登记全名，不做任何预处理或缩写
- 来源信息：保留原始格式，如会议名含年份则保留
- 所有信息按搜索结果原始状态录入
- **根据步骤2.4识别的载体标识，只收集对应载体类型的著录项目**

**必备项表格（初步素材草稿）：**

| 字段 | 类型要求 | 搜索结果信息 | 是否必备 |
|------|----------|--------------|----------|
| 主要责任者 | 有则必备 | 作者全名 | 是/否 |
| 题名 | 必备 | 论文标题 | 是 |
| 文献类型标识 | 必备 | [J]/[C]/[M]等 | 是 |
| 文献载体标识 | 纸质资源无需此项 | /OL等 | 是/否 |
| 刊名/会议名/出版者 | 必备 | 发表来源（含原始格式） | 是 |
| 年 | 必备 | 发表年份 | 是 |
| 卷/期/页码 | 有则必备 | 卷期页码 | 是/否 |
| 获取路径 | 纸质资源无需此项 | URL/DOI | 是/否 |
| 永久标识符 | 纸质资源有则必备 | DOI等 | 否 |
| 版本 | 预印本有则必备 | 版本号 | 是/否 |
| 创建或修改日期 | 电子资源有则必备 | YYYY-MM-DD | 是/否 |
| 学位授予单位 | 学位论文有则必备 | 单位名称 | 是/否 |
| 报告编号 | 报告有则必备 | 编号 | 是/否 |
| 标准编号 | 标准必备 | 标准号 | 是/否 |
| 专利申请号 | 专利有则必备 | 申请号 | 是/否 |

**重要说明：**
- 如果步骤2.4判断为不带/OL的纸质资源，则表格中"文献载体标识"、"获取路径"等电子资源特有字段应标记为"不适用"或省略
- 如果步骤2.4判断为带/OL的电子资源，则需包含完整的电子资源字段
- 具体著录项目对照 `references/citation-formats.md` 中对应文献类型的定义

**示例（BART论文 - 预印本，带/OL）：**

| 字段 | 搜索结果信息 |
|------|--------------|
| 主要责任者 | Mike Lewis, Yinhan Liu, Naman Goyal, Marjan Ghazvininejad, Abdelrahman Mohamed, Omer Levy, Ves Stoyanov, Luke Zettlemoyer |
| 题名 | BART: Denoising Sequence-to-Sequence Pre-training for Natural Language Generation, Translation, and Comprehension |
| 文献类型标识 | PP/OL |
| 出版平台 | arXiv |
| 创建日期 | 2019-10-29 |
| 获取路径 | https://arxiv.org/abs/1910.13461 |
| 永久标识符 | arXiv:1910.13461 |

**示例（期刊论文 - 纸质资源，不带/OL）：**

| 字段 | 搜索结果信息 |
|------|--------------|
| 主要责任者 | 张三, 李四, 王五 |
| 题名 | 某算法的研究与应用 |
| 文献类型标识 | J |
| 刊名 | 计算机学报 |
| 年 | 2023 |
| 卷(期) | 45(3) |
| 页码 | 123-135 |
| 永久标识符 | 10.3724/SP.J.1016.2023.00123 |

#### 2.7 常见问题排查

根据 `references/troubleshooting.md` 中的常见错误警示，**排查步骤2.6所得的元数据**：

| 检查项 | 检查内容 | 处理方式 |
|--------|----------|----------|
| 作者姓名 | 判断步骤2.6元数据中的主要责任者名字是否用汉语拼音表示；如果是汉语拼音则保留全称（姓首字母大写，名用全称），如果不是汉语拼音则姓首字母大写，名缩写；≤3个全部著录，>3个著录前3个加", et al."；禁止全大写 | 修正格式 |
| 版本处理 | 第1版可省略，其他版本用阿拉伯数字、序数缩写形式或其他标识表示；古籍版本照录 | 修正版本格式 |
| 年份重复 | 来源本身已含年份时，删除来源中的年份，保留后面年份 | 删除来源中的年份 |
| 类型路径一致性 | 文献类型标识与获取路径匹配 | 不匹配时降级处理 |
| 信息冗余 | 不添加类型要求外的信息 | 删除冗余信息 |

**排查对象说明：**
- 以步骤2.6收集的元数据为排查基准，因为这是最准确的搜索结果
- 不使用更早步骤中的初步或模糊数据

**排查结果处理：**
- 发现问题 → 修正并进入下一步
- 未发现问题 → 直接进入下一步

**排查修改后，输出规范化的必备项表格：**

| 字段 | 修改前 | 修改后 | 修改原因 |
|------|--------|--------|----------|
| 作者 | Mike Lewis, Yinhan Liu, Naman Goyal, et al. | Lewis M, Liu Yinhan, Goyal N, et al. | 汉语拼音全称，非汉语拼音名缩写，>3个加"et al." |
| 版本 | Third edition | 3rd ed | 用序数缩写形式表示 |
| 来源 | CVPR 2024 | CVPR | 删除来源中的年份 |

#### 2.8 生成引用格式

将纠错后的必备项填入格式模板，生成规范的引用格式。**输出这条引用：**

**顺序编码制示例：**
```
[1] Vaswani A, Shazeer N, Parmar N, et al. Attention is all you need[C/OL]//NIPS, 2017: 5998-6008. https://doi.org/10.48550/arXiv.1706.03762.
```

**该条文献处理完成。如有下一条文献，重新执行步骤2的所有子步骤。**

### 步骤3：汇总输出

当处理完所有参考文献后，汇总所有引用：

#### 3.1 纯文本引用格式

```
[1] 作者1. 题名1[类型标识]//来源1, 年份1: 页码1. 获取路径1.
[2] 作者2. 题名2[类型标识]//来源2, 年份2: 页码2. 获取路径2.
...
```

**注意：此步骤仅在所有文献逐条处理完成后执行。**

## 使用示例

### 示例1：单条文献验证

**场景：** 用户需要验证一篇会议论文的引用格式

**步骤执行：**
1. 分析指令 → 确认为单条文献
2. 提取关键字 → 论文标题、作者、会议名
3. 信息收集 → 搜索获取完整信息
4. 匹配验证 → 输出查证表格
5. 识别类型 → 确认为[C/OL]
6. 获取模板 → 输出会议录格式模板和必备项说明
7. 准备必备项 → 输出必备项表格
8. 排查问题 → 输出修改后表格
9. 生成引用 → 输出最终引用格式

**预期产出：**
- 查证表格
- 格式模板和必备项说明
- 必备项表格
- 修改后表格（如有问题）
- 最终引用格式

### 示例2：多条文献验证

**场景：** 用户需要验证3篇论文的引用格式

**步骤执行：**
1. 分析指令 → 确认为多条文献，制定处理顺序
2. 处理文献1 → 完整执行步骤2.1-2.8，输出所有中间结果
3. 处理文献2 → 完整执行步骤2.1-2.8，输出所有中间结果
4. 处理文献3 → 完整执行步骤2.1-2.8，输出所有中间结果
5. 汇总输出 → 输出3条文献的完整引用格式

**预期产出：**
- 每条文献：查证表格、格式模板、必备项表格、修改后表格、单条引用
- 最终：3条文献的完整引用格式列表

## 注意事项

- 处理时必须查阅参考文件，确保按类型要求著录
- 多条文献时必须逐条完整处理，禁止省略中间输出
- 作者著录：汉语拼音作者用全称，非汉语拼音姓全称名缩写；≤3个全部著录，>3个著录前3个加", et al."
- 优先采纳纸质资源而非电子资源
- 所有首字母大写，禁止全大写
